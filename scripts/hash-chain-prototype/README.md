# Hash-chain prototype (Phase 1 seed)

Working prototype for client-computed MemTree prefix hash chains — the
byte-identical JS replica of polychat's server-side hashing pipeline. See
`polychat/plans/2026-07-14_PLAN_client_hash_chain_delta_upload.md` for the
full plan; Appendix A there is the serialization contract these files
implement.

Verified 2026-07-14: reproduces the stored staging `messages_hash` for a real
183-message / 8MB conversation exactly, and matches the Python pipeline on an
adversarial fixture (floats, ensure_ascii escapes, astral-key sorting,
Python-whitespace stripping, system-reminder/tool_result quirks).

## Files

- `hash_parity.mjs` — the replica: Python-compatible canonical JSON
  (`json.dumps(x, separators=(',',':'), sort_keys=True)`, `ensure_ascii`),
  the preprocessing chain from `check_and_trigger_indexing`
  (`remove_cache_control` → `normalize_simple_user_messages` → pop system
  message → `strip_cc_system_reminders`), and incremental prefix hashing
  (`generate_messages_hash_sequence`). Run directly to emit the adversarial
  fixture (`hash_parity_fixture.json`) and its hash chain.
- `hash_incremental.mjs` — cross-turn incremental hasher: carries sha256
  state between turns (`Hash.copy()` forks), verifies history didn't change
  via native `JSON.stringify` compare keys, full recompute on rewrite
  (compaction). Benchmarked ~12–17ms warm / <100ms cold per turn on the 8MB
  conversation. Requires `real_convo.json` (below).

## Reproducing the parity check

JS side:

```bash
node hash_parity.mjs        # writes hash_parity_fixture.json, prints chain
```

Python ground truth (from a polychat checkout, polychat conda env):

```python
# hash_parity_ref.py — run with PYTHONPATH=<polychat checkout>
import json
from polychat.llm_api.extract_text import strip_cc_system_reminders
from polychat.llm_api.preprocess import normalize_simple_user_messages
from polychat.memory.history_job import (
    generate_messages_hash_sequence, remove_cache_control,
)
from polychat.utils import sys_msg_index

messages = json.load(open("hash_parity_fixture.json"))
remove_cache_control(messages)
messages = normalize_simple_user_messages(messages)
i = sys_msg_index(messages)
if i != -1:
    messages.pop(i)
print(json.dumps(generate_messages_hash_sequence(
    strip_cc_system_reminders(messages))))
```

The two outputs must be identical.

`real_convo.json` for `hash_incremental.mjs` is
`{"stored_hash": ..., "messages": [...]}` pulled from a real `chat_memory`
row (staging, via the DB tunnel — contains user data, never commit it).
